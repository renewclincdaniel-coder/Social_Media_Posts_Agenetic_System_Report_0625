# 社群貼文 Agentic 系統 · 運作報告（2026-06-25）

一份用 **HTML PPT** 呈現的系統運作說明，講清楚「社群貼文 Agentic 系統」如何從
**找對民眾真正在問的問題 → 對接主推療程 → 七天內容規劃與排程 → 人工核准發佈 → 每週自動回收成效**，
形成一個會越跑越準的閉環。

## 如何觀看

直接用瀏覽器打開 **`index.html`** 即可（純靜態，無需安裝）。

或啟動本機伺服器後瀏覽：

```bash
# 任選其一
python -m http.server 8080
npx serve .
```

然後開 <http://localhost:8080>。

## 鍵盤操作

| 按鍵 | 功能 |
| --- | --- |
| `←` `→` `Space` | 上一頁 / 下一頁 |
| `T` | 切換主題（內建多種配色） |
| `S` | 開啟**演講者模式**（current / next / 逐字稿 / 計時器） |
| `O` | 投影片總覽 |
| `F` | 全螢幕 |
| `N` | 講者備忘抽屜 |

> 每一頁都附有講者逐字稿（`<div class="notes">`），按 `S` 或 `N` 可叫出，內含不省略的細節說明。

## 內容大綱（15 頁）

1. 封面
2. 為什麼做這套系統（背景與痛點）
3. 全景：五階段閉環
4. 階段一｜選題：只寫民眾真正想解決的問題（CSV 真實需求／前 20 名）
5. 護欄一｜爬蟲優先人工、整理乾淨再上傳 CSV
6. 階段二｜匹配：把熱門話題對接主推療程
7. 護欄二｜韓國醫美貼文只當對照、不改寫
8. 階段三｜七天內容規劃：一個主題、七種切角
9. 發文節奏：一週兩次、引流／轉單交錯
10. 引流文 vs 轉單文
11. 階段四｜護欄三：人工核准才發佈
12. 階段五｜每週自動回收成效
13. 報告與飛輪：入庫 → AI 解讀 → 回饋知識庫
14. 一週節奏時間軸
15. 總結：三道護欄 + 帶來的價值
